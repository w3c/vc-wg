## 1	Preliminary remarks

- All major vocabularies are maintained using the same tool: [yml2vocab](https://github.com/w3c/yml2vocab/): the vocabulary is defined using a special `.yml` file, and the tool generates a human-readable HTML version, as well as formal “RDFS+bit of OWL” versions in Turtle and JSON-LD serializations. In all core cases the VC specifications include a (normative) hash of the JSON-LD versions (when appropriate).
- During When in development:
	- the YAML versions are part of the relevant GitHub repositories;
	- the HTML, JSON-LD, and Turtle versions appear only under the "w3c.github.io/…" URL-s: these files are generated, using yml2vocab, as part of GitHub actions;
	- In some cases there is also an SVG file, which is a hand-crafted visual representation of the vocabulary displayed in the HTML file. It is on the GigHub repository.
- JSON-LD context files are maintained manually on the relevant GitHub repositories. During development the canonical URL is redirected to a "w3c.github.io/…" URL.
- At the publication of the Recommendations the HTML, Turtle, and JSON-LD versions of the vocabularies, as well as the JSON-LD Context files, are copied to the W3C server under the https://www.w3.org/2025/credentials/ date-space folder. Redirections are set up from the canonical URLs of to the final places.
	- Where appropriate, the structure of the directories and files on `/2025/credentials` follow the versioning. This means that the hierarchy is prepared for new versions already.

(Note that the yml2vocab tool has evolved over the years, and it can now generate decent JSON-LD Context files as well. This could be used for newer vocabularies.)
